# AI 动态追踪器 (AI Pulse Tracker) 说明文档

## 1. 项目目标与定义

### 1.1 项目目标

本项目旨在开发一个智能的AI动态追踪系统，实时监测全球AI领域的最新动态、产品发布和技术突破，特别是来自硅谷等创新中心的信息。通过自动化的数据收集、AI分析和趋势预测，为用户提供全面、及时、有价值的AI行业洞察。

### 1.2 "AI动态追踪"的定义

"AI动态追踪"是指：

- **实时监测**：自动从多个权威来源获取AI相关的最新信息
- **智能分析**：利用AI技术对收集到的信息进行分类、评分和总结。**目前实现的主要是**对信息进行分类和评分
- **趋势识别**：通过数据分析识别AI行业的发展趋势和热点方向
- **个性化呈现**：根据用户需求和兴趣，提供定制化的内容展示

## 2. 主要功能

### 2.1 核心功能

1. **多源数据采集**
   - 自动从Hacker News、TechCrunch、MIT Tech Review等多个权威来源获取AI相关信息
   - 支持RSS订阅和API调用，确保信息的全面性和时效性
2. **AI驱动的内容分析**
   - 利用MiniMax API对文章进行智能分类（AI应用、AI基础设施、LLM模型）
   - 基于内容质量和来源权威性进行评分，筛选高质量内容
   - 自动生成文章摘要，方便快速浏览
3. **趋势分析报告**
   - 定期生成AI行业趋势分析报告
   - 从产品形态、技术能力、应用场景、商业化进展等多个维度进行分析
   - 提供专业、客观的行业洞察和预测
4. **用户友好的界面**
   - 响应式设计，适配不同设备
   - 分类标签系统，方便用户快速筛选感兴趣的内容
   - 直观的卡片式布局，展示文章标题、来源、评分和摘要
   - 一键刷新功能，获取最新内容

### 2.2 技术特点

- **自动化数据采集**：通过定时任务和手动触发，确保数据的及时更新
- **智能内容处理**：利用AI技术提高内容分析的准确性和效率
- **内存存储**：本地开发环境使用内存存储，确保快速响应
- **可扩展性**：模块化设计，易于添加新的数据源和分析功能

## 3. 开发过程与AI工具应用

### 3.1 开发步骤

#### 阶段一：项目初始化与基础架构搭建

1. **环境搭建**：使用Next.js 14+和TypeScript创建项目
2. **依赖管理**：配置Tailwind CSS和shadcn/ui组件库
3. **项目结构**：设计清晰的目录结构，分离前端和后端逻辑

#### 阶段二：数据采集与处理

1. **数据源配置**：集成Hacker News API和多个RSS源
2. **数据获取**：实现并行和顺序的数据获取策略，提高效率
3. **数据清洗**：去重和格式化处理，确保数据质量

#### 阶段三：AI分析与处理

1. **AI集成**：接入MiniMax API进行内容分析
2. **分类系统**：实现基于AI的文章分类功能
3. **评分机制**：设计综合评分系统，评估文章质量

#### 阶段四：前端开发与用户界面

1. **页面布局**：设计响应式布局，优化用户体验
2. **组件开发**：创建文章卡片、分类标签、刷新按钮等组件
3. **交互功能**：实现文章筛选、趋势分析展示等交互功能

#### 阶段五：部署与优化

1. **本地开发优化**：实现内存存储，提高开发效率
2. **部署配置**：准备Vercel部署环境
3. **性能优化**：优化数据获取和处理逻辑，提高响应速度

### 3.2 主要AI工具应用

1. **Qwen：主要是prompt编写**
2. **TRAE：代码实现**

## 4. 技术架构

### 4.1 系统架构

- **前端**：Next.js 14+、React、Tailwind CSS、shadcn/ui
- **后端**：Next.js API Routes、Node.js
- **数据存储**：内存存储（本地开发）、Upstash Redis（生产环境）
- **AI服务**：MiniMax API
- **数据源**：Hacker News API、多个RSS源

### 4.2 核心模块

1. **数据采集模块**：负责从多个来源获取AI相关信息
2. **数据处理模块**：对采集到的数据进行清洗、去重和格式化
3. **AI分析模块**：利用MiniMax API对内容进行分类、评分和摘要生成
4. **存储模块**：管理数据的存储和检索
5. **前端展示模块**：负责用户界面和交互功能

## 5. 使用指南

### 5.1 本地开发

1. 克隆项目仓库
2. 安装依赖：`npm install`
3. 配置环境变量（参考`.env.local.example`）
4. 启动开发服务器：`npm run dev`
5. 访问：<http://localhost:3000>

### 5.2 数据刷新

- **手动刷新**：点击页面上的"Refresh"按钮
- **自动刷新**：配置Vercel Cron作业（生产环境）

### 5.3 内容浏览

- **分类筛选**：点击页面顶部的分类标签，筛选不同类型的AI内容
- **质量排序**：默认按评分排序，展示高质量的AI相关文章
- **趋势分析**：查看页面顶部的趋势分析报告，了解AI行业最新动向
